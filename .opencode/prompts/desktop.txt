=== ARCHITECTURE ===
WA Desktop is a Composite Shell with:

MENUBAR: Dashboard | Company | Users | Templates | App Settings
  - Dashboard → WebView2 tab embedding wa-frontend (React SPA)
  - Company/Users/Templates → Management CRUD forms (MVP)
  - App Settings → Super Admin only (webhook, API keys)

SIDEBAR: WA Phone Number TreeView
  - Click any node → opens dedicated tab in Workspace

=== KEY CONVENTIONS ===
- Pure MVP (Passive View): Views are dumb, Presenters are the brain
- IView interfaces: NO WinForms types leaked (use primitives, DTOs, IList)
- IDisposable on all Presenters that subscribe to events
- Unidirectional data flow: UseCase mutates state, EventAggregator broadcasts
- Background tasks via Task.Run, UI marshal via InvokeIfRequired
- Snake_case JSON from Go backend → PascalCase C# DTOs
- WebView2: Evergreen mode (uses installed Edge)

=== ROLE-BASED MENUS ===
- Admin: Dashboard, Company, Users, Templates, WA Phone Numbers
- Super Admin: all of the above + App Settings
